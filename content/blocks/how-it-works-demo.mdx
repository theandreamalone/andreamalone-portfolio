## Why that restraint is the actual demo

It would be easy to build a version of this that generates more — writes fresh copy, invents new framing per visitor, sounds impressively adaptive. I didn't build that.

The harder, more useful problem was deciding where generation should stop, and building a system disciplined enough to hold that line while still being genuinely responsive to what you're asking. Composing the right answer from fixed, honest evidence is a harder design problem than generating something new — and it's the one that actually matters when the stakes are someone's hiring decision.

If you want to see it work: ask a real question below. Not a keyword — an actual thing you're trying to find out. That's the demo.
